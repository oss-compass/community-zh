开源世界每天都有数不尽的项目涌现，但如何准确衡量一个开源项目的健康度一直是个难题。幸运的是，现在我们有了开源项目的健康体检医生——OSS
Compass，最近它的 **开源生态评估体系与 SaaS 服务进行了全面的升级** ！让我们一起来看看都有哪些精彩的更新吧~

最近，OSS Compass 的官网文档页上公布了
全新的开源生态评估体系，通过这个全新的架构图，我们在“生产力、稳健性、创新力”和“软件、协作、人”交织的三维空间中构建了十几个评估模型。

![](media/2.png)

原先的 SaaS 平台只有 4 个评估模型：“协作开发指数”、“社区服务与支撑”、“社区活跃度”和“组织活跃度”，现在我们又新增了 3 个贡献者画像模型，包括“
**贡献者领域画像** ”、“ **贡献者里程画像** ”、“ **贡献者角色画像** ”。 **贡献者领域画像模型**

将贡献者为社区做出的贡献按照领域进行划分，可以看到观察类型贡献、Issue 类型贡献、Code 类贡献、论坛类贡献、聊天平台类贡献、媒体平台类贡献、活动类贡献和 Sponsor 贡献，统计的指标有 6 个，包括
**代码贡献者数量、代码贡献次数、Issue 贡献者数量、Issue 贡献次数、观察者数量、观察者贡献次数**。

<center>

![](media/3.png)![](media/4.png)

</center>

在 **贡献者角色画像模型**
中，在代码托管平台有管理行为的贡献者被称为管理者，没有管理行为的贡献者称为参与者，管理行为通常指 Issue 管理类贡献或 Code 管理类贡献。根据贡献者是否声明自己的组织身份，如是否通过公开的组织邮箱提交代码，把他们区分为组织或个人贡献者。

这样，角色画像模型把贡献者角色划分为 4 类：组织管理者、个人管理者、组织参与者、个人参与者，统计的指标有 4 个，
**包括组织贡献者数量、组织贡献次数、个人贡献者数量、个人贡献次数** 。

<center>

![](media/5.png)![](media/6.png)

</center>

**贡献者里程画像模型**
根据贡献者在社区的贡献量、贡献频率、贡献者类型，将贡献者里程划分 5 种状态：无状态、静默、访客、常客以及核心，统计的指标有 6 个，包括
**核心贡献者数量、核心贡献次数、常客贡献者数量、常客贡献次数、访客贡献者数量、访客贡献次数**。

<center>

![](media/7.png)![](media/8.png)

</center>

贡献者是推动开源社区发展的源动力所在。因此，了解开源项目中贡献者的情况对于开源社区管理来说是至关重要的。以上 3 个模型的统计指标都是我们经过大量洞察实践筛选出来的，相信能够给开源社区管理者们带来他们最需要了解的信息。

### 02 SaaS 服务升级：项目深度洞察功能上线

在 OSS Compass 官网的开源项目评估报告页，可以通过不同指标模型的曲线变化去感知社区的发展状况。最新升级中，我们的项目评估报告引入了
**项目深度洞察功能** ，变得更强大了！目前，深度洞察页面主要呈现了 **贡献者画像、Issue、Pull
Request** 这三个类型的数据。

在贡献者画像中，我们可以查看
**贡献者在代码托管平台的 ID 名、角色画像、里程画像、领域画像、贡献组织和贡献次数** 等详细信息，并且可以进行筛选和排序。Issue 和 Pull
Request 的详情页面则展示了每条 Issue 和 Pull
Request 的相关信息，包括标题、URL、状态、首次响应时长等，同时通过饼图展示了 Issue 和 Pull
Request 解决状态和评论数量分布情况。
总体来说，这次更新是开源社区管理者们的福音，能够给他们的开源社区治理工作带来诸多益处，也能够给让想要了解某一开源项目发展状况的人们一目了然及时获取相关信息。
